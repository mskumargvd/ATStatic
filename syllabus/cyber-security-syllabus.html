Cyber Security

3-Month Professional Program Syllabus

Month 1: The Defender's Foundation

Weeks 1 & 2: Security & Networking Fundamentals

Module 1.1: Core Security Principles

Objective: Understand the goals of cybersecurity.

Topics: The CIA Triad (Confidentiality, Integrity, Availability), Introduction to Risk Management, Overview of common threats (Malware, Phishing).

Module 1.2: The TCP/IP & OSI Models

Objective: Learn how data travels across the internet to identify anomalies.

Topics: The 7 Layers of the OSI Model, The 4 Layers of the TCP/IP Model, Encapsulation & Decapsulation.

Module 1.3: IP Addressing & Core Protocols

Objective: Understand network addressing and communication.

Topics: IPv4 Addressing, Public vs. Private IPs, Subnetting basics, TCP vs. UDP, DNS, DHCP.

Module 1.4: Hands-On Network Analysis

Objective: Gain practical experience analyzing live network traffic.

Topics: Introduction to Wireshark, Capturing and filtering packets, Following TCP streams.

Lab: Capturing your own web traffic and identifying the DNS, TCP, and HTTP protocols in action.

Week 3: Operating System Hardening

Module 2.1: Windows Security Fundamentals

Objective: Learn to secure the world's most common desktop OS.

Topics: The Windows File System (NTFS permissions), User Account Control (UAC), Local Security Policy, Windows Defender Firewall, Analyzing Windows Event Logs.

Module 2.2: Linux Security Fundamentals

Objective: Learn to secure the most common server OS.

Topics: The Linux File System (permissions: `rwx`), User and Group management, `sudo` privileges, Securing SSH, Basic firewall configuration with `ufw`/`firewalld`.

Lab: Harden a provided Linux virtual machine by creating a non-root user, configuring SSH key-based authentication, and setting up basic firewall rules.

Week 4: The Attacker's Mindset & Threat Intelligence

Module 3.1: The Cyber Kill Chain

Objective: Understand the structured phases of a typical cyberattack.

Topics: Reconnaissance, Weaponization, Delivery, Exploitation, Installation, Command & Control (C2), Actions on Objectives.

Module 3.2: The MITRE ATT&CK Framework

Objective: Learn the industry-standard language for describing attacker behavior.

Topics: Introduction to ATT&CK, understanding Tactics, Techniques, and Procedures (TTPs), Navigating the ATT&CK Matrix.

Module 3.3: Threat Intelligence

Objective: Learn how defenders use information to stay ahead of threats.

Topics: What is Cyber Threat Intelligence (CTI)? Indicators of Compromise (IoCs) vs. TTPs, Introduction to Open-Source Intelligence (OSINT).

Lab: Use the ATT&CK website to map the TTPs of a famous malware strain (e.g., WannaCry) to the framework.

Month 2: The Analyst's Toolkit

Weeks 5 & 6: Log Analysis with SIEM

Module 4.1: Introduction to SIEM

Objective: Understand how a SIEM aggregates and analyzes security data.

Topics: What is a SIEM? Overview of Splunk and the ELK Stack. The data pipeline (ingestion, parsing, normalization).

Module 4.2: Writing Basic & Intermediate Queries

Objective: Learn to search and filter massive amounts of log data.

Topics: Searching and filtering logs. Using statistical functions (`count`, `stats`).

Module 4.3: Correlation & Alerting

Objective: Learn to automatically detect suspicious patterns.

Topics: Creating correlation rules to detect attacks (e.g., brute force, impossible travel). Setting up automated alerts.

Lab: Ingest logs, write queries, and create a custom alert in a SIEM.

Week 7: Endpoint Detection & Response (EDR)

Module 5.1: The Need for EDR

Objective: Understand the role of modern endpoint protection.

Topics: Limitations of traditional antivirus. How EDR monitors process execution, file modifications, and network connections.

Module 5.2: Analyzing EDR Alerts

Objective: Learn to investigate suspicious activity on a computer.

Topics: Understanding process trees. Investigating an EDR alert from start to finish. Differentiating malicious vs. legitimate activity.

Lab: Analyze a simulated EDR alert for a malware infection, tracing its execution, persistence, and C2 communication.

Week 8: Vulnerability Management

Module 6.1: Identifying Vulnerabilities

Objective: Learn to proactively find weaknesses in systems.

Topics: The vulnerability management lifecycle. Using Nessus for credentialed and uncredentialed scans.

Module 6.2: Prioritizing & Reporting

Objective: Learn to assess risk and communicate findings.

Topics: Understanding the CVSS scoring system. Analyzing scan results and creating a prioritized remediation report.

Lab: Run a vulnerability scan against an unpatched VM and create a professional report on the top 3 critical findings.

Month 3: Incident Response & Career Prep

Week 9: Phishing Analysis

Module 7.1: Anatomy of a Phishing Email

Objective: Learn to deconstruct the most common attack vector.

Topics: Analyzing email headers to trace the origin. Identifying spoofed domains and suspicious language.

Module 7.2: Malicious Link & Attachment Analysis

Objective: Learn how to safely investigate payloads.

Topics: Using online sandboxing tools (like Any.Run) to safely open suspicious links and attachments.

Lab: Perform a full analysis of a real-world phishing email sample and document your findings in an incident report.

Week 10: Introduction to Digital Forensics

Module 8.1: Core Concepts

Objective: Understand the principles of evidence collection.

Topics: What is digital evidence? The importance of the Chain of Custody.

Module 8.2: Disk & Memory Forensics

Objective: Get an overview of forensic investigation techniques.

Topics: Introduction to disk imaging. Understanding what kind of artifacts can be recovered from memory (RAM).

Module 8.3: Hands-on with Autopsy

Objective: Gain experience with a professional forensics tool.

Topics: Introduction to the Autopsy digital forensics platform.

Lab: Use Autopsy to examine a sample disk image to recover deleted files and analyze user activity.

Weeks 11 & 12: Capstone Project & Interview Prep

Capstone Project: The SOC Analyst Simulation

Objective: Act as a Tier 1 SOC Analyst for a day.

Task: Students will be presented with a queue of 5-7 realistic security alerts in a simulated environment. For each alert, they must investigate, determine if it is a True Positive or a False Positive, document their findings in a professional incident ticket, and recommend clear remediation steps.

Career Preparation Module

Objective: Prepare for a successful job search.

Topics: Building a resume that highlights your new skills and projects. Optimizing your LinkedIn profile. A deep dive into common SOC Analyst interview questions (technical and behavioral). Building a home lab for continuous learning.
